# AI Interaction Diagnostic

Однофайловое FastAPI-приложение для объяснимой диагностики качества взаимодействия сотрудника с генеративным искусственным интеллектом.

Проект выполнен в рамках курсовой работы НИУ ВШЭ. Автор: Никита Усков, 2025–2026.

## Что делает сервис

Приложение анализирует рабочий диалог с генеративным ИИ и показывает, насколько пользователь:

- формулирует цель, контекст, ограничения и формат результата;
- запрашивает проверку фактов, допущений, рисков и альтернатив;
- вносит собственные данные, критерии и гипотезы;
- сохраняет контроль над итоговым решением, не передавая его полностью модели.

Результат — индивидуальный дашборд с интегральным баллом, четырьмя компонентами оценки, основаниями оценки и рекомендациями.

## Основные возможности

- анализ текста, вставленного вручную;
- загрузка диалогов в форматах `TXT`, `MD` и `PDF`;
- определение ролей по меткам `User:` / `AI:` и русским аналогам;
- автоматическое определение ролей в PDF без меток по расположению реплик слева и справа;
- OCR-распознавание PDF-сканов диалога;
- анализ публичных ссылок на чаты, если страница доступна серверу без авторизации и антибот-защиты;
- встроенный чат с сохранением истории сессии;
- история индивидуальных отчётов;
- выгрузка индивидуального отчёта в одностраничный PDF-дашборд;
- выгрузка командной аналитики в PDF;
- агрегированная командная аналитика при наличии не менее 10 отчётов;
- светлая и тёмная темы интерфейса;
- опциональная интеграция с GigaChat.

## Методика оценки

| Компонент | Содержание |
|---|---|
| `P` | Постановка задачи: цель, контекст, ограничения, формат, уточнения |
| `V` | Проверка результата: факты, источники, допущения, ограничения, альтернативы |
| `A` | Аналитическое участие: собственные данные, гипотезы, критерии, декомпозиция |
| `R` | Риск чрезмерного делегирования модели значимой части решения |

Интегральный показатель рассчитывается по формуле:

```text
I = 0.271P + 0.418V + 0.191A + 0.120(1 − R)
```

Весовые коэффициенты являются расчётным примером, используемым в учебной версии. Перед корпоративным внедрением они должны быть подтверждены экспертным оцениванием.

## Структура репозитория

```text
app.py              # приложение и встроенный интерфейс
requirements.txt    # зависимости Python
README.md           # описание и инструкция по запуску
.env                # пример настроек окружения
.gitignore          # исключения для Git
```

После первого запуска создаётся `ai_analyzer.db` — локальная SQLite-база с сессиями встроенного чата и сформированными отчётами. Она не загружается в GitHub.

## Быстрый запуск

Требуется Python 3.10 или новее.

### 1. Клонируйте репозиторий

```bash
git clone <адрес_вашего_репозитория>
cd ai-interaction-diagnostic
```

Либо скачайте репозиторий как ZIP и распакуйте его.

### 2. Создайте виртуальное окружение

```bash
python -m venv .venv
```

Windows:

```bat
.venv\Scripts\activate
```

macOS / Linux:

```bash
source .venv/bin/activate
```

### 3. Установите зависимости

```bash
pip install -r requirements.txt
```

### 4. Запустите приложение

```bash
python app.py
```

После запуска откройте:

```text
http://127.0.0.1:8000
```

Интерактивная документация API FastAPI:

```text
http://127.0.0.1:8000/docs
```

## OCR для PDF-сканов

Для PDF, в которых текст уже выделяется мышью, отдельная настройка OCR не нужна.

Для PDF-сканов дополнительно установите Tesseract OCR с языками `rus` и `eng`. После установки на Windows при необходимости создайте файл `.env` рядом с `app.py`:

```env
TESSERACT_CMD=C:\Program Files\Tesseract-OCR\tesseract.exe
OCR_LANGUAGE=rus+eng
```

В интерфейсе выберите, где расположены ваши сообщения в PDF: справа или слева. Сервис группирует распознанный текст в реплики и относит их к пользователю или ИИ по положению на странице.

## Анализ ссылки на чат

Сервис может обработать только публичную ссылку, если текст диалога доступен обычному HTTP-клиенту без авторизации, CAPTCHA и антибот-защиты.

Некоторые сервисы открывают ссылку в браузере, но блокируют автоматический серверный импорт. В частности, это может происходить с публичными страницами DeepSeek. В таком случае используйте экспорт или сохранение чата в PDF и загрузите его в сервис.

Приложение не обходит авторизацию, CAPTCHA и ограничения платформ.

## Встроенный чат и GigaChat

Анализ текста, файлов, PDF-отчёты и командная аналитика доступны без ключа GigaChat.

Для генерации ответов во встроенном чате установите дополнительную библиотеку:

```bash
pip install gigachat
```

Создайте файл `.env` рядом с `app.py`:

```env
GIGACHAT_API_KEY=ваш_ключ
GIGACHAT_MODEL=GigaChat:latest
```

Файл `.env` содержит секреты и не должен попадать в GitHub.

## Как пользоваться

### Индивидуальная диагностика

1. Откройте раздел **«Диагностика»**.
2. Укажите название команды, если отчёт нужно включить в командную аналитику.
3. Вставьте диалог, загрузите файл или укажите публичную ссылку.
4. Для PDF без подписей ролей выберите сторону, на которой находятся ваши сообщения.
5. Подтвердите отсутствие запрещённых данных.
6. Нажмите **«Проанализировать»**.
7. Изучите показатели `P`, `V`, `A`, `R`, основания оценки и рекомендации.
8. Нажмите **«Скачать отчёт в PDF»**, чтобы получить готовый одностраничный PDF-дашборд.

Для текстовых диалогов наиболее надёжный формат:

```text
User: Подготовь структуру аналитической записки...
AI: Предлагаю следующий вариант...
User: Проверь допущения, ограничения и альтернативные объяснения...
```

Допускаются русские метки: `Пользователь:`, `Сотрудник:`, `Ассистент:`, `Модель:`.

### Встроенный чат

1. Перейдите в раздел **«Встроенный чат»**.
2. Введите задачу и несколько уточнений.
3. Нажмите **«Проанализировать сессию»**.
4. Скачайте сформированный PDF-отчёт при необходимости.

### История отчётов

В разделе **«История отчётов»** доступны сохранённые индивидуальные отчёты. Для каждого отчёта можно открыть дашборд и скачать PDF.

### Командная аналитика

1. Создайте не менее 10 отчётов с одинаковым названием команды.
2. Откройте раздел **«Командная аналитика»**.
3. Укажите название команды.
4. Просмотрите средний балл, усреднённые компоненты, распределение уровней и рисков.
5. Нажмите **«Скачать командный отчёт в PDF»**.

## Ограничения и безопасность

- Не загружайте пароли, персональные данные, коммерческую тайну, платёжные сведения или клиентские документы.
- Анализирует только признаки, отражённые в тексте диалога. Действия пользователя за пределами переписки не учитываются.
- PDF-сканы и OCR могут содержать ошибки распознавания, поэтому для значимых задач следует проверять корректность извлечённого текста.
- Индивидуальный балл не является оценкой общей профессиональной эффективности сотрудника и не должен использоваться для кадровых санкций.
- Локальная версия не содержит авторизации, разграничения прав доступа и защищённого корпоративного контура.
